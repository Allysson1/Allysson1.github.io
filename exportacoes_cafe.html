Exportações de Café

De onde surgiu o projeto?

O projeto veio de um trabalho da faculdade na matéria de banco de dados, onde o objetivo era escolher uma quantidade massiva de dados para construir um Star Schema.

A origen dos dados

A origem dos dados vem de um dado .csv, com informações sobre exportações de produtos dos anos de 1997 à 1999.

Nos dados estão presentes o ano e o mês de sua transação, o código SH4 (número que se refere a determinado produto), o estado e o código do município que está exportando determinado produto, peso da mercadoria em KG e informações de valores do FOB (Free On Board, quer dizer que o exportador é responsável pela mercadoria até ela estar dentro do navio, para transporte, no porto indicado pelo comprador).

Agora que vimos de onde tudo veio, vamos começar a codar:
Antes de tudo, faço a importação do arquivo para o sql server.
Serão cinco tabelas para a construção de um star schema.
A primeira será a dimensão de tempo:

Create Table DimTempo
(CodigoTempo Int Identity(1,1) Not Null,
CodigoExportacao int not null,
Mes TinyInt Not Null,
MesPorExtenso Varchar(15) Not Null,
Quartil TinyInt Not Null,
QuartilPorExtenso Varchar(20) Not Null,
Ano Int Not Null,
AnoPorExtenso Varchar(40) Not Null
Constraint [PK_DimTime_TimeID] Primary Key Clustered (CodigoTempo))
Go

Usando Constraint, consigo criar uma chave primaria com o nome mais específico a coluna que quero utilizar como identificador único.
